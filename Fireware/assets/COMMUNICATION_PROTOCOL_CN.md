# TapBot 下位机通信协议说明

本文档面向 ROS2 上位机与下位机联调开发，整理当前固件中的两条通信链路：

- 主控板 <-> ROS2 上位机
- 主控板 <-> 末端副板

除特别说明外，所有多字节数据均采用小端格式，即低字节在前。

## 1. 基本数据类型

| 类型 | 长度 | 说明 |
|---|---:|---|
| `u8` | 1 byte | 无符号 8 位整数 |
| `s16` | 2 bytes | 有符号 16 位整数，小端 |
| `u16` | 2 bytes | 无符号 16 位整数，小端 |
| `s32` | 4 bytes | 有符号 32 位整数，小端 |
| `u32` | 4 bytes | 无符号 32 位整数，小端 |

常用缩放规则：

| 后缀 | 含义 | 示例 |
|---|---|---|
| `_x10` | 实际值乘以 10 后发送 | `123` 表示 `12.3` |
| `_x100` | 实际值乘以 100 后发送 | `9000` 表示 `90.00 deg` |
| `_x1000` | 实际值乘以 1000 后发送 | `50` 表示 `0.050` |

## 2. 主控板 <-> ROS2 上位机

### 2.1 串口参数

| 项目 | 参数 |
|---|---|
| 接口 | USART2 |
| 波特率 | 115200 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 最大 payload | 32 bytes |

### 2.2 帧格式

```text
+--------+--------+-------------+----------+----------+----------+
| 0xA5   | cmd_id | payload_len | payload  | crc_low  | crc_high |
+--------+--------+-------------+----------+----------+----------+
  1 byte   1 byte    1 byte       N bytes    1 byte     1 byte
```

字段说明：

| 字段 | 说明 |
|---|---|
| `0xA5` | 固定帧头 |
| `cmd_id` | 命令或反馈 ID |
| `payload_len` | payload 长度，范围 `0~32` |
| `payload` | 数据区 |
| `crc_low/crc_high` | CRC16-CCITT，小端 |

CRC 规则：

| 项目 | 参数 |
|---|---|
| 算法 | CRC16-CCITT |
| 初值 | `0xFFFF` |
| 多项式 | `0x1021` |
| 覆盖范围 | `cmd_id + payload_len + payload` |
| 是否包含帧头 | 不包含 |

### 2.3 上位机控制源

控制源枚举：

| 值 | 含义 |
|---:|---|
| `0` | 遥控器控制 |
| `1` | ROS2 上位机控制 |

上位机有效超时约为 `500 ms`。超过超时时间未收到有效上位机数据，主控板会回退到遥控模式，并对云台发送停止命令。

### 2.4 上位机 -> 主控板命令

#### `0x01 SET_SOURCE`

切换控制源。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `u8` | `source` | `0=遥控器`，`1=上位机` |

#### `0x02 SET_ALL`

综合控制命令。用于一次性下发钢带速度、末端舵机速度轴、云台速度方向等。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `s16` | `yushu_speed_x10` | 钢带速度命令，实际值 = 该值 / 10 |
| 2 | `s16` | `steer_axis_x10` | 末端三舵机遥控速度轴，实际值 = 该值 / 10 |
| 4 | `u8` | `steer_enable_mask` | bit0~bit3 对应舵机 0~3 使能 |
| 5 | `u8` | `ptz_ud_cmd` | 云台上下方向 |
| 6 | `u8` | `ptz_lr_cmd` | 云台左右方向 |
| 7 | `u8` | `ptz_ud_speed` | 云台上下速度 |
| 8 | `u8` | `ptz_lr_speed` | 云台左右速度 |
| 9 | `u8` | `clutch_cmd` | 可选，`0=断开`，`1=吸合`，未提供则无效 |

说明：该命令会退出钢带长度闭环，转为速度控制。

#### `0x10 YUSHU_SPEED`

钢带伸缩速度控制。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `s16` | `speed_x10` | 速度命令，实际值 = 该值 / 10 |
| 2 | `u8` | `clutch_cmd` | 可选，`0=断开`，`1=吸合` |

说明：该命令会退出钢带长度闭环。

#### `0x12 YUSHU_LENGTH`

钢带长度闭环控制。下位机使用编码器反馈进行 PID 闭环。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `s32` | `target_length_mm` | 目标长度，单位 mm |
| 4 | `u16` | `max_speed_x10` | 可选，最大速度，实际值 = 该值 / 10 |

如果未提供 `max_speed_x10`，下位机使用默认最大速度。

#### `0x11 EXEC_YUSHU_CAN`

底层玉树电机直通控制命令。当前工程配置下玉树电机主要走 USART1，CAN 直通用于兼容。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `u8` | `id` | 电机 ID |
| 1 | `u8` | `mode` | 电机模式 |
| 2 | `s16` | `W_x10` | 速度，实际值 = 该值 / 10 |
| 4 | `s16` | `T_x100` | 力矩，实际值 = 该值 / 100 |
| 6 | `s16` | `K_W_x1000` | 速度刚度，实际值 = 该值 / 1000 |

#### `0x20 STEER_SYNC`

末端舵机速度轴控制，适合连续控制。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `s16` | `steer_axis_x10` | 速度轴命令，实际值 = 该值 / 10 |
| 2 | `u8` | `enable_mask` | bit0~bit3 对应舵机 0~3 使能 |

#### `0x21 EXEC_STEER_SPEED`

末端旋转舵机速度直控。

格式 A：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `u8` | `enable_mask` | bit3 有效时读取旋转舵机速度 |
| 1 | `s16` | `speed0` | 舵机 0 速度 |
| 3 | `s16` | `speed1` | 舵机 1 速度 |
| 5 | `s16` | `speed2` | 舵机 2 速度 |
| 7 | `s16` | `speed3` | 舵机 3 速度 |

格式 B：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `u8` | `enable` | 非 0 时启用 |
| 1 | `s16` | `rotate_speed` | 旋转舵机速度 |

#### `0x22 EXEC_STEER_POSITION`

末端三个位置舵机位置控制。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `u8` | `enable_mask` | bit0~bit2 对应三个位置舵机 |
| 1 | `s16` | `pos0` | 舵机 0 目标位置 |
| 3 | `s16` | `pos1` | 舵机 1 目标位置 |
| 5 | `s16` | `pos2` | 舵机 2 目标位置 |
| 7 | `s16` | `pos3` | 可选，兼容字段 |
| 9 或 7 | `u16` | `run_time` | 运行时间 |
| 11 或 9 | `u16` | `speed` | 运行速度 |

说明：

- 当 payload 长度 >= 13 时，包含 `pos3`，`run_time` 从 offset 9 开始。
- 当 payload 长度 >= 11 且 < 13 时，不含 `pos3`，`run_time` 从 offset 7 开始。

#### `0x23 EXEC_STEER_MODE`

末端舵机模式设置。当前主板实现会设置为默认混合模式。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `u8` | `mode` | 当前未进一步区分，仅触发默认模式 |

#### `0x30 PTZ`

云台速度控制命令，进入周期控制流程。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `u8` | `up_down_cmd` | 上下方向 |
| 1 | `u8` | `left_right_cmd` | 左右方向 |
| 2 | `u8` | `up_down_speed` | 上下速度 |
| 3 | `u8` | `left_right_speed` | 左右速度 |

方向枚举：

| 值 | 含义 |
|---:|---|
| `0` | 停止 |
| `1` | 上 |
| `2` | 下 |
| `3` | 左 |
| `4` | 右 |

#### `0x31 EXEC_PTZ`

云台速度控制命令，立即执行并发送云台控制帧。

Payload 同 `0x30 PTZ`。

#### `0x32 EXEC_PTZ_ANGLE`

云台角度闭环控制。下位机结合 IMU yaw/pitch 反馈进行 PID 控制。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `s16` | `yaw_deg_x100` | 目标 yaw，单位 deg x100 |
| 2 | `s16` | `pitch_deg_x100` | 目标 pitch，单位 deg x100 |
| 4 | `u16` | `tolerance_x100` | 角度容差，单位 deg x100 |
| 6 | `u8` | `speed` | 最大速度，0 表示使用默认值 |

#### `0x7F HEARTBEAT`

上位机心跳。

Payload：空。

### 2.5 主控板 -> 上位机反馈

主控板反馈使用同一 `0xA5` 帧格式。

#### `0x80 ARM_STATE`

周期：约 100 ms。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `u16` | `seq` | 反馈序号 |
| 2 | `u32` | `time_ms` | 下位机反馈时间戳 |
| 6 | `u8` | `active_source` | 当前实际控制源 |
| 7 | `u8` | `system_state` | 系统状态，当前固定为 0 |
| 8 | `u16` | `fault_flags` | 故障标志位 |
| 10 | `s32` | `length_mm` | 钢带编码器长度，单位 mm |
| 14 | `s16` | `line_speed_mm_s` | 钢带线速度，单位 mm/s |
| 16 | `u8` | `limit_switch` | 限位开关状态 |
| 17 | `u8` | `clutch_state` | 离合器输出状态 |
| 18 | `s16` | `servo_pos0` | 舵机 0 目标/缓存位置 |
| 20 | `s16` | `servo_pos1` | 舵机 1 目标/缓存位置 |
| 22 | `s16` | `servo_pos2` | 舵机 2 目标/缓存位置 |
| 24 | `s16` | `rotate_servo_speed` | 旋转舵机速度 |
| 26 | `s16` | `imu_yaw_x100` | IMU yaw |
| 28 | `s16` | `imu_pitch_x100` | IMU pitch |
| 30 | `u8` | `imu_valid` | IMU 有效标志 |

#### `0x81 ACTUATOR_ECHO`

周期：约 100 ms。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `s16` | `yushu_speed_x10` | 当前钢带速度命令 |
| 2 | `s16` | `cmd_W_x10` | 电机底层 W |
| 4 | `s16` | `cmd_T_x100` | 电机底层 T |
| 6 | `s16` | `cmd_KW_x1000` | 电机底层 K_W |
| 8 | `u8` | `motor_mode` | 电机模式 |
| 9 | `s16` | `steer_speed0` | 舵机 0 速度 |
| 11 | `s16` | `steer_speed1` | 舵机 1 速度 |
| 13 | `s16` | `steer_speed2` | 舵机 2 速度 |
| 15 | `s16` | `steer_speed3` | 舵机 3 速度 |
| 17 | `u8` | `ptz_ud_cmd` | 云台上下方向 |
| 18 | `u8` | `ptz_lr_cmd` | 云台左右方向 |
| 19 | `u8` | `ptz_ud_speed` | 云台上下速度 |
| 20 | `u8` | `ptz_lr_speed` | 云台左右速度 |
| 21 | `u8` | `encoder_valid` | 编码器有效标志 |
| 22 | `u8` | `steer_mode` | 末端舵机模式 |
| 23 | `u8` | `ptz_status` | 云台状态 |

#### `0x82 DIAG`

周期：约 500 ms。  
该帧通过 50 ms 反馈调度槽发送，避免 USART2 DMA 帧互相覆盖。

Payload：

| Offset | 类型 | 字段 | 说明 |
|---:|---|---|---|
| 0 | `u8` | `rmt_online` | 遥控器在线 |
| 1 | `u8` | `upper_online` | 上位机在线 |
| 2 | `u8` | `encoder_valid` | 编码器有效 |
| 3 | `u8` | `imu_valid` | IMU 有效 |
| 4 | `u8` | `reserved` | 保留 |
| 5 | `u16` | `limit_switch` | 限位开关状态 |
| 7 | `u16` | `fault_flags` | 故障标志 |

故障标志：

| Bit | 含义 |
|---:|---|
| 0 | 上位机通信超时 |
| 1 | 遥控器掉电或通信超时 |
| 2 | CAN1 / 编码器通信超时 |
| 3 | 云台状态错误 |
| 4 | CAN2 / IMU 通信超时或 IMU 无效 |

## 3. 主控板 <-> 末端副板

### 3.1 串口参数

| 项目 | 参数 |
|---|---|
| 接口 | UART5 / RS485 |
| 波特率 | 115200 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 最大 payload | 32 bytes |

### 3.2 帧格式

```text
+--------+---------+--------+-------------+----------+----------+----------+
| 0xA6   | node_id | cmd_id | payload_len | payload  | crc_low  | crc_high |
+--------+---------+--------+-------------+----------+----------+----------+
  1 byte   1 byte    1 byte    1 byte       N bytes    1 byte     1 byte
```

字段说明：

| 字段 | 说明 |
|---|---|
| `0xA6` | 固定帧头 |
| `node_id` | 当前固定为 `0x01` |
| `cmd_id` | 命令或反馈 ID |
| `payload_len` | payload 长度，范围 `0~32` |
| `payload` | 数据区 |
| `crc_low/crc_high` | CRC16-CCITT，小端 |

CRC 规则：

| 项目 | 参数 |
|---|---|
| 算法 | CRC16-CCITT |
| 初值 | `0xFFFF` |
| 多项式 | `0x1021` |
| 覆盖范围 | `node_id + cmd_id + payload_len + payload` |
| 是否包含帧头 | 不包含 |

### 3.3 主控板 -> 副板命令

| ID | 名称 | Payload |
|---:|---|---|
| `0x01` | `HEARTBEAT` | 空 |
| `0x02` | `ESTOP` | 空 |
| `0x03` | `STOP` | 空 |
| `0x04` | `SET_MODE` | `u8 mode` |
| `0x20` | `SERVO123_SPEED` | `s16 speed0`, `s16 speed1`, `s16 speed2` |
| `0x21` | `SERVO123_POSITION` | `s16 pos0`, `s16 pos1`, `s16 pos2`, `u16 run_time`, `u16 speed` |
| `0x30` | `AUTO_START` | `u16 forward_ms`, `u16 reverse_ms`, `s16 speed` |
| `0x31` | `AUTO_STOP` | 空 |
| `0x40` | `PUSHROD` | 已定义，当前主板发送侧尚未封装 |

副板模式：

| 值 | 含义 |
|---:|---|
| `0` | 遥控速度模式 |
| `1` | 上位机位置模式 |
| `2` | 自动任务模式 |
| `3` | 急停模式 |

### 3.4 副板 -> 主控板反馈

| ID | 名称 | Payload |
|---:|---|---|
| `0x80` | `ACK` | `u8 state`, `u8 status`, `u8 ack_cmd_id` |
| `0x81` | `STATE` | `u8 state`, `u8 error` |
| `0x82` | `SERVO` | 已定义，当前主板未解析 |
| `0x83` | `ERROR` | `u8 error` |

主板当前会保存最近一次 ACK 对应的命令 ID，可通过 `SubBoard_LinkGetLastAckCmd()` 获取。

副板状态：

| 值 | 含义 |
|---:|---|
| `0` | 未知 |
| `1` | 空闲 |
| `2` | 遥控模式 |
| `3` | 上位机模式 |
| `4` | 自动任务运行中 |
| `5` | 完成 |
| `6` | 错误 |
| `7` | 急停 |

副板错误码：

| 值 | 含义 |
|---:|---|
| `0` | 无错误 |
| `1` | CRC 错误 |
| `2` | 长度错误 |
| `3` | 超时 |
| `4` | 传感器错误 |
| `5` | 推杆错误 |
| `6` | 舵机错误 |

## 4. 上位机开发建议

1. 每条上位机命令必须按帧格式封装 CRC。下位机只处理 CRC 正确的帧。
2. 上位机进入控制模式后，应以小于 500 ms 的周期发送命令或心跳。
3. 云台位置控制建议使用 `0x32 EXEC_PTZ_ANGLE`，由下位机基于 IMU 做闭环。
4. 钢带长度控制建议使用 `0x12 YUSHU_LENGTH`，由下位机基于编码器做闭环。
5. 停止运动时，上位机应主动发送速度 0 或切回遥控源；下位机也有超时保护，但不应依赖超时作为正常停车手段。
6. 上位机解析反馈时，应按 `cmd_id` 区分 `ARM_STATE`、`ACTUATOR_ECHO` 和 `DIAG`，不要假设反馈帧固定交替到达。
